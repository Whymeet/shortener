{# Пошаговый гайд подключения домена. Используется на дашборде (с плейсхолдером домена) и на странице домена (с реальным доменом). TLS выпускается автоматически (Caddy on-demand) — ручной настройки сервера (nginx/certbot) больше нет. #} {% macro steps(domain, server_ip) %}
  1. Шаг 1. DNS. В панели регистратора домена (где покупали {{ domain }}) создайте A-запись, которая направит домен на наш сервер: 
    Имя (Host):  @     (или {{ domain }})
Тип:         A
Значение:    {{ server_ip or "IP вашего сервера" }}
    Подождите распространения (обычно минуты, иногда до часа) и проверьте — команда должна вернуть IP сервера: 
    dig +short {{ domain }}     →  {{ server_ip or "IP сервера" }}
  2. Шаг 2. Активация. Когда A-запись указывает на сервер, нажмите «Отметить активным» вверху страницы домена. Дальше ничего делать не нужно: HTTPS-сертификат для {{ domain }} выпустится автоматически при первом заходе по короткой ссылке (первый запрос может занять пару секунд, дальше — мгновенно). Ни перезапуска сервиса, ни ручной настройки сервера.

Статус «активен» — это разрешение и на автовыпуск сертификата, и на создание коротких ссылок. Деактивация останавливает создание новых ссылок.

{% endmacro %}
